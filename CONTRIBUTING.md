# Contributing to AI-Enhanced Interactive Book Agent

Thank you for your interest in contributing to the AI-Enhanced Interactive Book Agent! We appreciate your time and effort.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/book-agent.git`
3. Create a virtual environment and install dependencies (see README.md)
4. Create a feature branch: `git checkout -b feature/your-feature-name`

## Development Workflow

1. Make changes to the codebase
2. Add or update tests as needed
3. Run the tests to ensure everything passes
4. Commit your changes with a descriptive commit message
5. Push to your fork
6. Submit a pull request

## Code Style

- Follow PEP 8 guidelines for Python code
- Use type hints for all function signatures
- Write docstrings for all public functions and classes
- Keep functions and methods focused and small
- Use meaningful variable names

## Testing

- Write unit tests for new functionality
- Ensure all tests pass before submitting a pull request
- Aim for high test coverage

## Commit Messages

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests after the first line

## Pull Requests

- Keep pull requests focused on a single topic
- Include a clear description of the changes
- Reference any related issues
- Include tests when adding new functionality or fixing bugs

## Questions?

If you have any questions, feel free to open an issue or contact the maintainers.

Thank you again for your contribution!
